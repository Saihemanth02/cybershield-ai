Setup Lab

Isolated Target Sandbox Setup

Construct your safe laboratory environment locally on VirtualBox or VMware. Do NOT test offensive tools outside these parameters.

1

Configure Host-Only Network Adapter

Open VirtualBox > Tools > Network Manager. Click Create. Ensure it is named vboxnet0 and assign a static IPv4 address. Leave the DHCP server disabled to avoid random address assignment.

# CLI Verification of active VirtualBox host-only interfaces
VBoxManage hostonlyif ipconfig vboxnet0 --ip 192.168.56.1 --netmask 255.255.255.0
2

Set Dual-Network Adapters on Attack Node

Ensure your Kali Virtual Machine has two network interfaces: Adapter 1 mapped to NAT (for package updates) and Adapter 2 mapped to Host-Only Adapter (for target scanning).

1

Verify Interface Allocation

Boot Kali Linux and query terminal adapters to verify that Adapter 2 has successfully bound to the host-only subnet.

# Query networking devices and verify IP addressing
ip addr show eth1
2

Perform Stealth Scan Verification

Check if you can ping or scan within the subnet using Nmap stealth flags.

# Scan host-only segment to verify targets
nmap -sP 192.168.56.0/24
1

Run Target Container

Pull and launch the OWASP Juice Shop vulnerable target inside your target VM (Ubuntu/Debian) bound strictly to the Host-Only interface on port 3000.

# Pull and run Juice Shop container inside sandbox
docker run -d -p 192.168.56.10:3000:3000 bkimminich/juice-shop
2

Decoupled Web proxy intercept configuration

Launch Burp Suite on Kali, assign browser settings to proxy queries to port 8080, and verify traffic intercept flows when querying http://192.168.56.10:3000.
